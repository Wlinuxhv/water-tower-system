## 核心架构变更

### 变更说明
- **旧架构**: STC8G 从机控制水泵继电器
- **新架构**: ESP8266 主机直接控制水泵继电器，STC8G 只做传感器采集

### 变更原因
1. **集中控制**: 所有控制逻辑集中在 ESP8266，便于统一管理
2. **安全性**: 主机可以直接监控和控制，避免从机故障导致失控
3. **简化从机**: STC8G1K08 资源有限 (8KB Flash, 1KB RAM)，只做传感器更稳定
4. **响应更快**: 主机直接控制继电器，减少通信延迟

### 硬件连接变更

#### ESP8266 主机 (新增继电器控制)
```
ESP8266 GPIO -> 三极管 -> 继电器 -> 水泵
  - D0 (GPIO16): 水泵继电器控制
  - 5V 电源：继电器线圈供电
  - 续流二极管：保护三极管
```

#### STC8G1K08 从机 (移除继电器)
```
STC8G1K08:
  - P1.0 (AIN0): 液位传感器 ADC 输入
  - P3.4: 缺水传感器数字输入
  - P2.0-P2.3: PAN3031 LoRa 通信
  - 移除继电器驱动电路
```

## 系统需求

### 功能需求

#### FR-1: 水位监测
- **描述**: 实时监测各水塔水位
- **负责**: STC8G1K08 从机
- **频率**: 每 5 秒上报一次
- **精度**: 0-100% (ADC 转换)

#### FR-2: 缺水保护
- **描述**: 检测井水是否缺水
- **负责**: STC8G1K08 从机检测，ESP8266 主机决策
- **响应**: 缺水时立即关闭所有水泵

#### FR-3: 水泵控制
- **描述**: 自动控制水泵启停
- **负责**: **ESP8266 主机直接控制**
- **逻辑**:
  - 水位 < 20%: 开启水泵
  - 水位 > 90%: 关闭水泵
  - 缺水时：禁止开启

#### FR-4: 远程监控
- **描述**: 手机 APP 远程查看和控制
- **负责**: ESP8266 Web 服务器
- **接口**: REST API + WebSocket

#### FR-5: 本地显示
- **描述**: OLED 实时显示状态
- **负责**: ESP8266 主机
- **内容**: 水位、模式、WiFi 状态

### 性能需求

#### PR-1: 响应时间
- 传感器数据上报：≤ 5 秒
- 水泵控制响应：≤ 1 秒
- APP 控制响应：≤ 2 秒

#### PR-2: 可靠性
- 系统可用性：≥ 99%
- 数据丢失率：≤ 1%
- 看门狗复位：超时 2 秒自动复位

#### PR-3: 功耗
- 从机待机电流：≤ 10mA
- 主机待机电流：≤ 100mA
- 水泵工作时：≤ 500mA

## 通信协议

### LoRa 数据帧格式

#### 传感器数据 (从机 -> 主机)
```
[Byte0: NodeID][Byte1: CMD_SENSOR][Byte2: Len][Byte3: WaterLevel][Byte4: WellWaterOK]
```

#### 控制命令 (主机 -> 从机)
```
[Byte0: HostID][Byte1: NodeID][Byte2: CMD][Byte3: Len][Byte4+: Data]
```

**注意**: 水泵控制命令 (CMD_PUMP_CTRL) 现在由 ESP8266 直接通过 GPIO 执行，不再通过 LoRa 发送给从机。

### 命令定义
| 命令 | 代码 | 方向 | 说明 |
|------|------|------|------|
| CMD_HEARTBEAT | 0x01 | 双向 | 心跳包 |
| CMD_SENSOR_DATA | 0x02 | 从->主 | 传感器数据 |
| CMD_READ_SENSOR | 0x03 | 主->从 | 读取传感器 |
| ~~CMD_PUMP_CTRL~~ | ~~0x04~~ | ~~主->从~~ | ~~水泵控制 (已废弃)~~ |
